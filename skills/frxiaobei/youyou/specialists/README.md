# 专科专家会诊系统使用指南

## 系统概述

本系统通过多个专科的 Skill（技能）和 Subagent（子代理）来模拟真实的多学科专家会诊（MDT），帮助您全面分析医疗数据。

## 核心组件

### 1. 专科 Skill 定义（.claude/specialists/）

每个专科都有一个独立的 Skill 定义文件，包含：
- 角色定义和专业领域
- 分析重点和关注指标
- 分析原则和安全红线
- 输出格式规范
- 示例分析用语

**支持的13大专科+1个协调器：**
- `cardiology.md` - 心内科
- `endocrinology.md` - 内分泌科
- `gastroenterology.md` - 消化科
- `nephrology.md` - 肾内科
- `hematology.md` - 血液科
- `respiratory.md` - 呼吸科
- `neurology.md` - 神经内科
- `oncology.md` - 肿瘤科
- `orthopedics.md` - 骨科 [NEW]
- `dermatology.md` - 皮肤科 [NEW]
- `pediatrics.md` - 儿科 [NEW]
- `gynecology.md` - 妇科 [NEW]
- `psychiatry.md` - 精神科/心理科
- `general.md` - 全科（协调员）

### 2. 会诊协调器（consultation-coordinator.md）

负责：
- 识别需要哪些专科参与会诊
- 并行启动多个专科 subagent
- 整合各专科意见
- 生成综合会诊报告

### 3. Slash Commands

#### `/consult` - 多学科专家会诊

```bash
# 分析所有数据
/consult all

# 分析最近5条记录
/consult recent 5

# 分析指定日期
/consult date 2025-12-31

# 分析日期范围
/consult date 2025-12-01 to 2025-12-31

# 默认分析最近3条
/consult
```

#### `/specialist` - 单专科咨询

```bash
# 查看支持的专科
/specialist list

# 咨询心内科
/specialist cardio recent 3

# 咨询内分泌科
/specialist endo all

# 咨询肿瘤科
/specialist onco date 2025-12-31
```

## 工作流程

### 会诊流程

```
用户输入 /consult
    ↓
读取医疗数据
    ↓
识别异常指标
    ↓
确定参与专科
    ↓
并行启动专科 subagent
    ↓  ↓  ↓
心内科 内分泌科 消化科 ...
    ↓  ↓  ↓
  各专科分析
    ↓  ↓  ↓
    会诊协调器
    ↓
整合意见 → 生成报告
```

### 单专科咨询流程

```
用户输入 /specialist cardio
    ↓
读取心内科 Skill 定义
    ↓
读取医疗数据
    ↓
启动心内科 subagent
    ↓
生成心内科分析报告
```

## 安全红线（严格遵守）

所有专科专家都严格遵守以下原则：

### ❌ 禁止行为
1. **不给出具体用药剂量**
   - ×: "服用阿托伐他汀 20mg"
   - √: "建议咨询医生调整降脂药物"

2. **不直接开具处方药名**
   - ×: "开具阿司匹林肠溶片"
   - √: "建议咨询医生是否需要抗血小板治疗"

3. **不判断生死预后**
   - ×: "预后差，生存期短"
   - √: "建议积极治疗，定期复查评估"

4. **不替代医生诊断**
   - ×: "确诊为冠心病"
   - √: "提示可能存在冠心病风险，建议心内科进一步检查"

### ✅ 允许行为
- 解读医疗检验指标的临床意义
- 识别异常指标和潜在风险
- 提供健康生活方式建议
- 推荐针对性检查项目
- 协助制定随访计划
- 整合多学科专家意见

## 使用示例

### 示例1：初次会诊

假设您刚做完体检，有多项检查结果：

```bash
/consult all
```

系统会：
1. 读取所有检查数据
2. 识别异常指标
3. 自动邀请相关专科（如心内科、内分泌科、消化科等）
4. 并行分析
5. 生成综合报告，包括：
   - 各专科分析
   - 优先级排序
   - 综合建议
   - 随访计划

### 示例2：专项深入分析

如果血脂异常特别明显，想深入咨询心内科：

```bash
/specialist cardio all
```

心内科专家会：
1. 详细分析血脂各项指标
2. 评估心血管风险
3. 提供针对性的饮食运动建议
4. 建议复查时间和项目

### 示例3：定期复查后对比

复查后想看变化趋势：

```bash
/consult recent 10
```

系统会分析最近10条记录，观察：
- 指标变化趋势
- 治疗效果评估
- 调整管理建议

## 报告格式

### 会诊报告包含：

1. **病例摘要** - 数据概览
2. **各专科分析** - 每个专科的独立分析
3. **综合评估** - 关键问题排序（紧急/重要/常规）
4. **综合建议** - 生活方式、检查计划、专科会诊建议
5. **健康提醒** - 需警惕的症状、监测重点
6. **随访计划** - 复查时间和预期目标
7. **重要声明** - 免责声明

### 单专科报告包含：

1. **数据概览** - 主要异常指标
2. **详细分析** - 按系统/指标分类分析
3. **风险评估** - 风险等级和因素
4. **建议** - 饮食、运动、监测、就医建议

## 最佳实践

### 1. 定期会诊
建议每次有新的检查结果后，运行 `/consult` 更新综合评估

### 2. 专项深入
对于特别关注的异常，使用 `/specialist` 获取深入分析

### 3. 趋势观察
使用 `recent` 参数分析多条记录，观察趋势

### 4. 按需咨询
根据具体问题选择合适的专科咨询

## 技术架构

```
Claude Code
    ↓
Slash Commands (/consult, /specialist)
    ↓
Subagent 系统
    ├─> 专科 Skill 定义 (专家知识库)
    ├─> 会诊协调器 (并行调度 + 意见整合)
    └─> 医疗数据 (data/*.json)
    ↓
分析报告 (Markdown 格式)
```

## 注意事项

1. **数据质量**：确保检查单识别准确，数据完整
2. **隐私保护**：所有数据存储在本地，不上传云端
3. **理性使用**：报告仅供参考，不替代医生诊断
4. **及时就医**：如有紧急症状，立即就医

## 新增专科（2025-12-31）

### 骨科
**专科代码**：`ortho`

**擅长领域**：
- 骨折与骨损伤
- 关节炎（骨关节炎、类风湿关节炎、痛风）
- 骨质疏松症
- 运动损伤
- 脊柱疾病

**关注指标**：
- 骨代谢：钙、磷、维生素D、ALP、骨密度
- 炎症：CRP、ESR、尿酸
- 影像：X线、CT、MRI、骨密度

**使用场景**：
- 骨折术后复查
- 关节疼痛评估
- 骨质疏松筛查

### 皮肤科
**专科代码**：`derma`

**擅长领域**：
- 湿疹、皮炎
- 痤疮
- 银屑病
- 皮肤感染
- 皮肤肿瘤筛查

**关注指标**：
- 过敏：IgE、嗜酸性粒细胞
- 炎症：CRP、白细胞
- 检查：皮肤镜、病理活检

**使用场景**：
- 过敏性皮肤病
- 痤疮治疗
- 皮肤肿瘤筛查

### 儿科
**专科代码**：`pedia`

**擅长领域**：
- 儿童生长发育
- 新生儿疾病
- 营养性疾病
- 儿童感染
- 疫苗接种

**关注指标**：
- 生长发育：身高、体重、骨龄（**年龄特异性参考值**）
- 营养：血常规、铁、锌、维生素D
- 感染：CRP、白细胞

**使用场景**：
- 儿童保健评估
- 发育迟缓筛查
- 营养指导

### 妇科
**专科代码**：`gyne`

**擅长领域**：
- 月经疾病
- 妇科炎症
- 子宫肌瘤、卵巢囊肿
- 宫颈癌筛查
- 更年期综合征

**关注指标**：
- 性激素：FSH、LH、E2、P、T、PRL
- 肿瘤标志物：CA125、CA19-9
- 筛查：HPV、TCT

**使用场景**：
- 月经不调
- PCOS评估
- 宫颈癌筛查
- 更年期管理
